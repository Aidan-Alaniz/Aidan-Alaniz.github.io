## Overview

Banterbird is a full-stack web application built as part of the ASCTE 301/401 curriculum. Modeled on Twitter, it allows users to create posts and view a live feed of content. The backend is written in Python and connects to external APIs to extend the app's functionality beyond what could be built from scratch.

This was the first project where I had to manage both sides of a web application simultaneously — the frontend that users see and the backend server logic that handles data — and make them work together.

## Features

- User post creation and dynamic feed display
- Python REST-style backend handling requests and data
- External API integration for extended functionality
- Web-based frontend with a clean user interface

## Tools & Skills Used

- **Python** — backend logic, request handling, and API integration
- **HTML/CSS** — frontend structure and layout
- **REST APIs** — connecting to external services to extend app functionality
- **Full-stack thinking** — managing the interface between client and server

## What I Learned

This was my first real full-stack project and the learning curve was steep in a useful way. The biggest shift was understanding that the frontend and backend are two separate systems that need to agree on how they communicate — data format, endpoints, timing. Getting that wrong meant nothing worked, and debugging across both layers at once taught me how to isolate problems more systematically. Working with external APIs also showed me how much you can build by composing existing services rather than reinventing everything from scratch.
